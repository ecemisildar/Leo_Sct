### Supervisor for full_rotate
**Purpose:** Trigger a full rotation when the robot is likely stuck near walls (i.e., multiple obstacles detected), to help reorient toward the center.

States:
  - name=0, initial=True, marked=True
  - name=1, initial=False, marked=True

Events:
  - obstacle_front, obstacle_left, obstacle_right, path_clear, full_rotate

Transitions:
  # In state 0, detect if surrounded by obstacles (front+left or front+right), then allow full_rotate
  - source=0, target=1, event=obstacle_front
  - source=1, target=1, event=obstacle_left
  - source=1, target=1, event=obstacle_right
  - source=1, target=0, event=full_rotate
  # If path is clear, stay in state 0 (no need to rotate)
  - source=0, target=0, event=path_clear
  - source=1, target=0, event=path_clear

### Supervisor for counterclockwise_turn
**Purpose:** Occasionally trigger a counterclockwise turn when the path is clear for a long time to avoid straight wall-following and encourage center drift.

States:
  - name=0, initial=True, marked=True
  - name=1, initial=False, marked=True

Events:
  - obstacle_front, obstacle_left, obstacle_right, path_clear, counterclockwise_turn

Transitions:
  # Accumulate path_clear events to trigger a turn
  - source=0, target=1, event=path_clear
  - source=1, target=1, event=path_clear
  - source=1, target=0, event=counterclockwise_turn
  # If obstacle detected, reset to state 0 (turn not needed)
  - source=0, target=0, event=obstacle_front
  - source=0, target=0, event=obstacle_left
  - source=0, target=0, event=obstacle_right
  - source=1, target=0, event=obstacle_front
  - source=1, target=0, event=obstacle_left
  - source=1, target=0, event=obstacle_right

### Supervisor for move_backward

  - name=1, initial=False, marked=True

  - source=0, target=0, event=path_clear

  - source=1, target=0, event=obstacle_right
